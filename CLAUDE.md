# DataDecoded

## Project Overview
A concept revision website for data analysts and data scientists.
Built with plain HTML, CSS, JavaScript. Deployed on Render.

## Personal Links
- LinkedIn: https://www.linkedin.com/in/saibysani18/
- Always add LinkedIn URL in the footer of every page

## Design System
- Colors: Navy #0A192F, White #FFFFFF, Electric Blue #0077FF
- Font: Inter from Google Fonts
- Every concept page must have a "Key Takeaway" box in blue
- Dark mode is the default. Light mode toggle must be available 
  on every page via a sun/moon icon in the top right navbar
- Dark mode colors: Background #0A192F, Text #FFFFFF, 
  Accent #0077FF
- Light mode colors: Background #F8F9FA, Text #1A1A2E, 
  Accent #0077FF
- Save user's mode preference in localStorage so it 
  persists across pages
- SQL category icon must be a clean database cylinder SVG 
  icon in electric blue — never use emoji for category icons
- Statistics icon: clean line chart SVG in electric blue
- Python icon: clean snake or code bracket SVG in electric blue

## Writing Style
- Plain English, no jargon without explanation
- Always include a real world example
- End every concept with a Key Takeaway box
- Writing should sound like Sai Bysani - conversational, simple, direct
- Use human language only - no AI buzzwords, no "delve", no "unleash", 
  no "harness", no "game-changer", no "revolutionary"
- Write like a senior data professional explaining to a smart colleague
- Use textbook accuracy but interview-friendly tone
- Short sentences. No fluff. Every word earns its place.
- If a concept can be explained in 3 lines, don't write 10
- Use industry terms correctly but always explain them on first use

## Folder Structure
- /sql/ for all SQL concepts
- /statistics/ for all statistics concepts  
- /python/ for all Python concepts
- styles.css is the single CSS file - never create separate CSS files

## Rules
- Never use React, Vue or any framework
- Never create inline styles - always use styles.css
- Every new concept page must follow the same template
- Mobile responsive always

## Git Rules
- After completing any task, automatically stage all 
  changes, write a descriptive commit message, and push 
  to GitHub
- Commit message format: "feat: [what was built]" or 
  "fix: [what was fixed]"
