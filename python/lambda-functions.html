Lambda Functions

Anonymous one-liner functions for quick operations

What Is a Lambda?

A lambda is a small, anonymous function defined in a single line. It takes arguments, performs one expression, and returns the result - all without needing a def statement or a name.

Real-World Analogy

A regular function is like a recipe card you label and file away - it has a name, and you pull it out whenever you need it. A lambda is like telling someone a quick instruction on the spot: "Just double it." You do not bother writing it down because you only need it right now.

The Syntax

Python lambda arguments: expression

Compare a regular function to a lambda:

Python # Regular function def double(x): return x * 2 # Lambda equivalent double = lambda x: x * 2 # Both work the same way: double(5) # returns 10

When to Use Lambdas

Lambdas shine when you need a small throwaway function as an argument to another function. The most common cases in data work:

Sorting with a custom key:

Python names = ["Carol", "Bob", "Alice"] sorted(names, key=lambda x: len(x)) # ['Bob', 'Carol', 'Alice'] - sorted by name length

With map() to transform data:

Python prices = [10, 20, 30] with_tax = list(map(lambda p: p * 1.08, prices)) # [10.8, 21.6, 32.4]

With filter() to select data:

Python numbers = [1, 2, 3, 4, 5, 6, 7, 8] big = list(filter(lambda x: x > 5, numbers)) # [6, 7, 8]

In pandas for quick column operations:

Python import pandas as pd df = pd.DataFrame({"name": ["alice", "bob"], "salary": [50000, 60000]}) df["name"] = df["name"].apply(lambda x: x.title()) # Capitalizes names: "Alice", "Bob"

When NOT to Use Lambdas

  • Complex logic - if you need multiple lines, use def
  • Reuse - if you will call the function in multiple places, give it a proper name with def
  • Readability - if the lambda is hard to read at a glance, a named function is better
Python # Bad - hard to read lambda result = sorted(data, key=lambda x: (x['dept'], -x['salary'], x['name'].lower())) # Better - named function def sort_key(employee): return (employee['dept'], -employee['salary'], employee['name'].lower()) result = sorted(data, key=sort_key)

Key Takeaway

Lambdas are throwaway one-liner functions. Use them when you need a small function as an argument (sorting keys, map/filter, pandas apply). If the logic needs multiple lines or you will reuse it, write a regular function with def instead.
