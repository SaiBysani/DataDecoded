JOINs Explained

Combining tables - the right way, every time

What Is a JOIN?

A JOIN combines rows from two or more tables based on a related column between them. It is how you connect data that lives in separate tables.

Real-World Analogy

Think of two spreadsheets: one has customer names and IDs, the other has order details and customer IDs. A JOIN is like doing a VLOOKUP - you match rows from both sheets using the customer ID to see the full picture.

INNER JOIN

Returns only rows that have a match in both tables.

SQL SELECT c.name, o.order_id, o.amount FROM customers c INNER JOIN orders o ON c.customer_id = o.customer_id;

Result: Alice and Bob (they exist in both tables). Carol, Dave, and the guest order are excluded.

Use when: you only want records that have data on both sides.

LEFT JOIN

Returns all rows from the left table, plus matching rows from the right. Non-matches get NULL.

SQL SELECT c.name, o.order_id, o.amount FROM customers c LEFT JOIN orders o ON c.customer_id = o.customer_id;

Result: All 4 customers appear. Carol and Dave show NULL for order columns because they never ordered.

Use when: you want all customers regardless of whether they have orders (e.g., finding customers who have never purchased).

RIGHT JOIN

Returns all rows from the right table, plus matching rows from the left. Non-matches get NULL.

SQL SELECT c.name, o.order_id, o.amount FROM customers c RIGHT JOIN orders o ON c.customer_id = o.customer_id;

Result: All 3 orders appear. The guest order (customer_id NULL) shows NULL for the customer name.

Use when: you want all orders even if the customer record is missing. In practice, most people rewrite RIGHT JOINs as LEFT JOINs by swapping the table order.

FULL OUTER JOIN

Returns all rows from both tables. Non-matches on either side get NULL.

SQL SELECT c.name, o.order_id, o.amount FROM customers c FULL OUTER JOIN orders o ON c.customer_id = o.customer_id;

Result: All 4 customers and all 3 orders. Carol and Dave have NULL orders; the guest order has a NULL customer name.

Use when: you want the complete picture from both sides, missing matches included.

Key Takeaway

INNER JOIN = only matches. LEFT JOIN = all from the left, matches from the right. In real work, LEFT JOIN is the most common because you usually have a "main" table you want to keep complete while enriching it with data from another table.
