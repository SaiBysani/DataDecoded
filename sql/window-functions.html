Window Functions

Calculations across rows - without losing your detail

What Are Window Functions?

A window function performs a calculation across a set of rows that are somehow related to the current row. Think of it as looking through a "window" of related rows while keeping every individual row in your result.

The key difference from GROUP BY: window functions do not collapse your rows. You get the calculation and the original detail, side by side.

Real-World Analogy

Imagine you are in a classroom and the teacher posts everyone's test score on the board. With GROUP BY, you would only see the class average - one number. With a window function, you see every student's score plus the class average next to each name. Everyone keeps their row; the aggregate rides along.

When to Use Them

  • Ranking rows - who is #1, #2, #3 in each department?
  • Running totals - cumulative revenue over time
  • Comparing to group statistics - how does this sale compare to the department average?
  • Accessing other rows - what was last month's value? (LAG/LEAD)

Example: Ranking Employees by Salary

Suppose you have an employees table and you want to rank each employee within their department by salary.

SQL SELECT name, department, salary, RANK() OVER ( PARTITION BY department ORDER BY salary DESC ) AS salary_rank FROM employees;

Here is what this produces:

namedepartmentsalarysalary_rank
AliceEngineering130,0001
BobEngineering120,0002
CarolEngineering120,0002
DaveMarketing95,0001
EveMarketing88,0002

Notice: every row is preserved. RANK() assigned rankings within each department (the PARTITION BY clause). Bob and Carol share rank 2 because they have the same salary.

The Syntax Pattern

SQL function_name() OVER ( PARTITION BY column -- defines the "window" (group) ORDER BY column -- defines the ordering within the window )

Common window functions: RANK(), DENSE_RANK(), ROW_NUMBER(), SUM(), AVG(), LAG(), LEAD().

Key Takeaway

Window functions let you add aggregate-level insights (ranks, running totals, averages) to every row without collapsing your data. If you need the detail AND the summary, reach for a window function.
