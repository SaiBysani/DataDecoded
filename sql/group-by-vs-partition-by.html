GROUP BY vs PARTITION BY

Same grouping idea, very different results

The One-Sentence Difference

GROUP BY collapses rows into one row per group. PARTITION BY keeps every row and adds group-level calculations alongside them.

Real-World Analogy

Imagine a class of 30 students across 3 sections. GROUP BY gives you one summary row per section (like a report card summary: "Section A average = 85"). PARTITION BY gives you all 30 rows but writes the section average next to each student so you can compare individuals against their section.

The Data

Let's say you have a sales table:

salespersonregionamount
AliceEast500
BobEast300
CarolWest700
DaveWest400

GROUP BY - Collapsed Summary

SQL SELECT region, SUM(amount) AS total_sales FROM sales GROUP BY region;
regiontotal_sales
East800
West1100

4 rows collapsed into 2. Individual salesperson detail is gone.

PARTITION BY - Detail + Summary

SQL SELECT salesperson, region, amount, SUM(amount) OVER (PARTITION BY region) AS region_total FROM sales;
salespersonregionamountregion_total
AliceEast500800
BobEast300800
CarolWest7001100
DaveWest4001100

All 4 rows are preserved. The region total rides alongside each row.

Side-by-Side Comparison

FeatureGROUP BYPARTITION BY
Rows in resultOne per groupAll original rows
Used withAggregate functions (SUM, COUNT...)Window functions (OVER clause)
Access to detailLost (columns must be in GROUP BY or aggregated)Preserved (every column available)
Best forSummary reportsEnriching detail with group stats

When to Use Which

  • Need a summary table (total revenue by region)? Use GROUP BY.
  • Need each row plus its group context (each sale with region total)? Use PARTITION BY.
  • Need to rank, number, or compare rows within groups? Use PARTITION BY with a window function.

Key Takeaway

GROUP BY summarizes and collapses. PARTITION BY enriches without collapsing. If you want the detail AND the group calculation, PARTITION BY (via a window function) is your tool. If you just want the summary, GROUP BY is simpler and sufficient.
